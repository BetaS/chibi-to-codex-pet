## Context

현재 `CodexPetBuilder`는 검증된 ZIP을 생성한 뒤 blob 다운로드 링크를 제공하고, 링크 클릭 시 GA4 다운로드 이벤트만 기록한다. 사용자가 이미 결과물을 받은 뒤에만 요청을 노출해야 하며, 브라우저는 실제 파일 저장 완료를 알려주지 않으므로 다운로드 링크 클릭을 성공 시점으로 사용한다.

## Goals / Non-Goals

**Goals:**

- ZIP 다운로드를 시작한 직후에만 간단한 GitHub Star 안내를 표시한다.
- 다운로드 자체를 차단하거나 Star를 강제하지 않는다.
- 한 빌더 작업 세션에서 반복 노출하지 않는다.
- 키보드, 스크린 리더, 작은 화면에서도 명확하게 사용할 수 있다.

**Non-Goals:**

- GitHub Star를 자동으로 추가하거나 사용자의 Star 여부를 확인하지 않는다.
- GitHub API를 호출해 실시간 Star 수를 표시하지 않는다.
- 영구 쿠키나 `localStorage`를 사용해 사용자를 추적하지 않는다.
- 페이지 진입 또는 Pet 생성 시작 시점에는 안내를 표시하지 않는다.

## Decisions

### 다운로드 링크 클릭을 노출 시점으로 사용

검증된 ZIP 링크의 `click` 처리에서 기존 분석 이벤트를 유지하고 안내 상태를 연다. 생성 직후 노출하는 대안은 아직 가치를 전달받기 전 흐름을 방해하고, OS 저장 완료 감지는 브라우저에서 신뢰할 수 없어 제외한다.

### 컴포넌트 수명 동안 한 번만 표시

`CodexPetBuilder`의 ref로 노출 여부를 기억한다. 새 영구 저장소나 식별자를 만들지 않으면서 같은 결과 링크를 반복 클릭해도 다시 열리지 않는다. 페이지 새로고침 뒤에는 다시 표시될 수 있다.

### 종속성 없는 접근 가능한 모달 사용

별도 UI 라이브러리 없이 `role="dialog"`, `aria-modal`, 제목 연결, Escape 및 명시적 닫기 버튼, 초기 포커스와 닫은 뒤 포커스 복원을 구현한다. 배경은 현재 앱의 어두운 색상과 보라색 강조색을 재사용하고, 장식적 doodle 요소는 사용하지 않는다.

### 저장소 링크만 외부로 이동

주요 동작은 `https://github.com/BetaS/chibi-to-codex-pet`을 새 탭으로 열며 `rel="noreferrer"`를 사용한다. GitHub가 제공하지 않는 자동 Star 동작처럼 오해시키지 않도록 버튼 문구는 GitHub에서 Star할 수 있다는 의미로 번역한다.

## Risks / Trade-offs

- [실제 파일 저장 완료가 아니라 링크 클릭을 측정함] → 문구와 테스트에서 다운로드 시작 이후의 안내로 정의한다.
- [모달이 작업 흐름을 가릴 수 있음] → 다운로드 동작 이후에만 표시하고 즉시 닫을 수 있게 하며 이후 반복 노출하지 않는다.
- [Star 여부를 알 수 없어 이미 Star한 사용자에게도 보일 수 있음] → 상태 확인을 위한 GitHub 인증이나 추적을 추가하지 않고 부담이 적은 단일 안내로 제한한다.
